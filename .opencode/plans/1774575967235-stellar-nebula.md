## 1.12 Hook manuel pre-push

### Objectif
Créer un hook Git local qui intercepte les pushes contenant des fichiers >5Mo non trackés par Git LFS.

### Fichier à créer

**`.git/hooks/pre-push`** (exécutable):
```bash
#!/bin/bash
# Hook pre-push : interdit le push de fichiers >5 Mo non gérés par Git LFS

echo "🔍 Vérification pre-push : contrôle des fichiers volumineux hors LFS..."

THRESHOLD=5242880  # 5 Mo en octets
VIOLATIONS=""

# Parcourir les fichiers modifiés ou ajoutés
while IFS= read -r -d '' file; do
  if [ -f "$file" ]; then
    SIZE=$(stat -c%s "$file" 2>/dev/null || stat -f%z "$file" 2>/dev/null)
    if [ -n "$SIZE" ] && [ "$SIZE" -gt "$THRESHOLD" ]; then
      FILTER=$(git check-attr filter "$file" 2>/dev/null | awk '{print $3}')
      if [ "$FILTER" != "lfs" ]; then
        SIZE_MB=$(echo "scale=1; $SIZE / 1048576" | bc)
        VIOLATIONS="$VIOLATIONS\n  - $file (${SIZE_MB} Mo)"
      fi
    fi
  fi
done < <(git diff --cached --name-only --diff-filter=ACM -z 2>/dev/null)

if [ -n "$VIOLATIONS" ]; then
  echo ""
  echo "❌ Push refusé : fichiers >5 Mo non gérés par Git LFS :"
  echo -e "$VIOLATIONS"
  echo ""
  echo "   Solution : git lfs track \"*.<extension>\""
  echo "              git add .gitattributes"
  echo "              git add <fichier>"
  echo "              git commit -m \"chore: track <extension> with Git LFS\""
  echo ""
  exit 1
fi

echo "✅ Contrôle pre-push réussi : tous les fichiers volumineux sont dans LFS."
exit 0
```

### Commandes d'exécution

```bash
chmod +x .git/hooks/pre-push
```

### Note importante
- Le fichier `.git/hooks/pre-push` n'est PAS tracké par Git (car `.git/` n'est jamais commité)
- Ce hook doit être recréé manuellement sur chaque clone du repo
- Alternative: utiliser un template de hooks global avec `git config core.hooksPath`

## 1.13 Configuration des GitHub Secrets

### Script à créer

**`scripts/setup_secrets.sh`**:
```bash
#!/bin/bash
# Script d'aide à la configuration des GitHub Secrets
# À exécuter UNE SEULE FOIS après avoir créé le dépôt

echo "📋 Configuration des GitHub Secrets pour le pipeline CI/CD"
echo "============================================================"
echo ""
echo "Exécute les commandes suivantes en remplaçant les valeurs :"
echo ""
echo "gh secret set HF_TOKEN          # Token Hugging Face (write access)"
echo "gh secret set ANTHROPIC_API_KEY # Clé API Anthropic pour génération email"
echo "gh secret set SMTP_SERVER       # Ex: smtp.gmail.com"
echo "gh secret set SMTP_PORT         # Ex: 587"
echo "gh secret set SMTP_USERNAME     # Ton adresse email"
echo "gh secret set SMTP_PASSWORD     # Mot de passe d'application Gmail"
echo "gh secret set MAIL_TO           # Email(s) des membres de l'équipe"
echo ""
echo "Vérification après configuration :"
echo "gh secret list"
```

### Commandes

```bash
chmod +x scripts/setup_secrets.sh
git add scripts/setup_secrets.sh
git commit -m "chore: add secrets setup helper script"
git push origin develop
```

## Ordre des opérations

1. Créer `.git/hooks/pre-push` avec le contenu ci-dessus
2. `chmod +x .git/hooks/pre-push`
3. Créer `scripts/setup_secrets.sh` avec le contenu ci-dessus
4. `chmod +x scripts/setup_secrets.sh`
5. Commit et push sur `develop`

```bash
# Vérifier que le hook est exécutable
ls -la .git/hooks/pre-push

.git/hooks/pre-push

# Vérifier le script secrets
```

## Secrets nécessaires pour le CI/CD

| Secret | Description |
|--------|-------------|
| `HF_TOKEN` | Token Hugging Face (accès write) |

| `SMTP_PORT` | Port SMTP (ex: 587) |
| `SMTP_USERNAME` | Adresse email expéditrice |
| `SMTP_PASSWORD` | Mot de passe d'application |
| `MAIL_TO` | Email(s) destinataires |
| `HF_SPACE_URL` | URL du HuggingFace Space |
